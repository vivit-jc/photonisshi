# コンセプト
- 写真撮影により簡単に日誌を作るアプリ
- 写真を撮った時間帯により、自動で画像にタグを付与する（開店時、閉店時等）
- webアプリだが、スマホで使うことを想定している。写真撮影以外の機能はPCからでも使える。
- 現場の利用者向けの子アプリとしての機能と、管理側の親アプリとしての機能がある。
- 本ソフトは試作であるため、本番時に必要な機能の一部はオミットする

# 環境
- vue3
- DBはsupabase、画像ストレージはsupabase storage
- ホスティングはnetlify
- スマホで起動した際、カメラと連動するようにする

# 画像アップロードについて
- 画像は長辺が800pxになるよう圧縮する

# タグ
- タグは、画像に複数付けられる。
- タグは子アプリで任意に増やせる他、親アプリで「共通タグ」を設定するとすべての子アプリにそのタグが表示されるようになる。
- 写真撮影時、スマホのGPSにより現在地を取得し、あらかじめ設定した「GPSタグ」の範囲内であれば、そのタグが付けられる。
- GPSタグは、例えば「◯◯店」のように建物等として利用する。あらかじめ親アプリから緯度・経度を入力しておき、そこから300m以内程度であればそのタグを付ける。

# 親アプリ
## ダッシュボード
- 子アプリを使用するユーザーを指定し、そのタイムラインを参照することができる。
- 子アプリに対してメッセージやスタンプなどが送れる
- タイムラインは、最大3列まで同時に表示できる。それぞれで期間やタグなどで絞り込みができる。

# 子アプリ
## アクセス時
- ページにアクセスした時、ユーザーはダイアログにユーザー名を入力しOKを押す。
- そのユーザーが既に存在すればそのユーザーのページを開き、そうでなければ新規ユーザーページに移動する
- ユーザー名入力欄は、一度入力した値を記憶しておき２回目以降はそのままOKを押すだけにする

## ダッシュボード
- 「カメラを起動」ボタンがあり、それを起動し撮影することで、その日の日誌の写真記録を追加する（複数枚可）
- 「写真を貼る」ボタンがあり、これは既にスマホ内にある写真を選び追加する機能である。
- 写真を追加する際、コメントを追加することもできる。
- ダッシュボードではコメントが記入できる。コメントはテキスト欄に入力しコメントボタンを押すと書き込まれる。コメントには書き込み時間も記録され、表示される。
- 写真やコメントは長押しで削除できる。削除する際は確認ダイアログを表示する。
- ダッシュボードには、その日の写真やコメントだけが表示される。

## タイムライン
- タイムラインは、デフォルトで過去7日間の写真やコメントが表示される。
- 期間やタグなどを設定して絞り込みができる。
